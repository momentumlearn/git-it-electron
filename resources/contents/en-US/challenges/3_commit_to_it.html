Create a file in your new repository, add something to that file and commit that change with Git.

Commits

Commits are core to using Git. They are the moments in which you save and describe the work you've done. They are the ticks in the timeline of your project's history.

Create a New File

Now that you've got a repository started, let's add a file to it.

Create a new empty file inside the 'hello-world' directory you created in the last challenge. We can use the touch command to do this from the command line.

touch readme.txt

Now open the file in your code editor and write a little bit of text (perhaps type "Hello World!") and save the file. To open the current working directory in VS Code, you can run this command:

code .

Check Status

Next check the status of your repository to find out if there have been changes. You know you have changed something, but does Git?

Make sure you're still within your 'hello-world' directory when you're running these commands.

Check the status of your repo:

git status

This command doesn't change anything; it only reports on what's happening in your repo.

You will see the path of the file that has been created, along with some other information about the repo.


      On branch main
      No commits yet
      Untracked files:
        (use "git add <file>.." to include in what will be committed)
          readme.txt
      nothing added to commit but untracked files present (use "git add" to track)

Add Changes

You should still be within your 'hello-world' directory for this step.

Now that we've made some changes, let's tell Git that we want to include them in a snapshot of changes for this repo.

The add command means mark these changes to be included in the next commit.

To add the file you just created so that it becomes a part of the changes you will commit (aka save) with Git, run the add command followed by the path of the changed file:

git add readme.txt

Run git status again after you have run this command to confirm that your changes have been added.

Commit Changes

Now that we have marked our file for committing, we are ready to actually make the commit.

You should still be within your 'hello-world' directory for this step.

To commit means to save a snapshot of changes at this point in the project. It's our first commit, so it's adding an entire new file and its contents. When we create a commit, we'll include a brief message that describes the reason the commit is being made.

This command will commit the changes to the repo's history and allow you to write a short message describing the updates.

git commit -m "Create readme"

Make More Changes

Now add another line to 'readme.txt', and save the file again in your text editor.

In terminal, you can view the difference between the file now and how it was at your last commit.

Tell Git to show you the diff:

git diff

Now with what you just learned above, add and commit this latest change.

Click the 'Verify' button to confirm that you have committed the changes to your repository.
