Git

Git is free open source software (the source code is public) written by Linus Torvalds who also wrote the Linux operating system's kernel.

Git is a program for keeping track of changes over time, known in programming as version control. If you've used a "track changes" feature in a text editing software then you're already familiar with the concept!

Confirm Git Configuration

In the computer set-up instructions, we asked you to install Git using Homebrew, and to configure your name, email, and a few default settings.

In this step we'll check that configuration.

Open your terminal. We're using iTerm2 as our terminal program at Momentum.

You do this by using command-spacebar to open the Spotlight Search bar that is built into Mac OS, and type "iterm".

You can verify that Git is available at your command line by typing:

git --version

This will return the version of Git on your computer and look something like this:

git version 2.40.0

The set-up instructions had you configure Git so it is able to associate your work to you. Check to see if it is configured properly:

Check to see if your name is configured in Git.

This command should print out your name:

git config --global user.name

If it's missing or incorrect, set it now with this command:

git config --global user.name "Your Name"

Check to see if your email is configured in Git.

This command should print out your email:

git config --global user.email

If it's missing or incorrect, set it now with this command:

git config --global user.email "your_email@youremail.com"

Check to see if your default branch is configured in Git.

This command should show that your default branch is set to main:

git config --global init.defaultBranch

If it's missing or incorrect, set it now with this command

git config --global init.defaultBranch main

Check to see if a default merge strategy is configured in Git.

This command should show that the rebase merge strategy is set to false:

git config --global pull.rebase

If it's missing or incorrect, set it now with this command

git config --global pull.rebase false

You're done with your first challenge! Click the 'Verify' button to check the challenge.

Dollar Signs in Code Documentation

Dollar signs $ are often used in programming documentation to signify that the line is command line code (see the code snippets above). You don't actually type the $ in though; only type what comes after.
